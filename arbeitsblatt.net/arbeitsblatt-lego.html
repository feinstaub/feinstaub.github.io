Python mit dem NXT verbinden

Hinweise:

  • Das "$" gehört nicht zum Befehl, sondern sagt, dass das nachfolgende Kommando in der bash-Shell eingegeben werden soll.

  • Das ">>>" gehört nicht zum Befehl, sondern sagt, dass man sich hier in der Python-Shell befindet.

Python = Python3

Arbeitsverzeichnis erzeugen:

$ cd ~/tmp/pynxt/

Python-nxt-Bibliothek herunterladen:

$ git clone https://github.com/eelviny/nxt-python

(Falls git nicht installiert ist, dann vorher $ sudo zypper install git oder alternativ über den Download-Knopf)

In das heruntergeladene Verzeichnis wechseln.

$ cd nxt-python

Python-nxt-Bibliothek für eigenen Nutzer (im Gegensatz zu systemweit) installieren:

$ python setup.py install --user

Ausprobieren, ob das geklappt hat:

$ python >>> import nxt

Wenn keine Fehlermeldung erscheint, war die Installation erfolgreich.

USB konfigurieren

Die nächsten Schritte basieren auf dieser Anleitung: https://github.com/Eelviny/nxt-python/wiki/Installation

$ sudo zypper install python3-usb

$ sudo groupadd lego

$ sudo usermod -a -G lego <username>

(<username> kann mit $ whoami herausgefunden werden)

Leere Datei anlegen:

$ sudo touch /etc/udev/rules.d/70-lego.rules

Schauen, ob die Datei da ist:

$ ls -la /etc/udev/rules.d/70-lego.rules

Rechte setzen:

$ sudo chmod u+w /etc/udev/rules.d/70-lego.rules

Datei mit kwrite bearbeiten:

$ kdesu kwrite /etc/udev/rules.d/70-lego.rules

Folgendes eintragen (eine Zeile):

Speichern und kwrite wieder schließen.

Wieder zurück ins Arbeitsverzeichnis:

$ cd ~/tmp/pynxt/

Beispiele kopieren:

$ git clone https://github.com/Eelviny/nxt-python-examples

Troubleshooting

  • Brick: Orangene Taste drücken.

  • nxt.locator.find_one_brick() findet nichts

    • Restart udev: sudo udevadm control --reload-rules http://unix.stackexchange.com/questions/39370/how-to-reload-udev-rules-without-reboot

  • siehe auch https://github.com/Eelviny/nxt-python/issues/72 → TODO: try again and report back

  • $ udevadm monitor

    • Zeigt an, wenn USB-Geräte angeschlossen oder entfernt werden.

    • Den angezeigen Pfad kann man im folgenden Befehl verwenden:

  • $ sudo udevadm info --path PFAD

  • PFAD ist z. B. /devices/pci0000:00/0000:00:1a.0/usb1/1-1/1-1.2
