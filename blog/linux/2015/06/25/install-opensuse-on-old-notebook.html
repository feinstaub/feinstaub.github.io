openSUSE 13.2 auf altem Notebook installieren

25 Jun 2015

Notizen zur Installation von openSUSE 13.2 auf einem alten Medion-Notebook.

IST-Zustand war:

  • Medion Laptop mit Windows XP, das seit 2014 aus der Wartung gelaufen ist.

SOLL-Zustand:

  • Dual-Boot mit vorhandenem Windows XP und openSUSE 13.2. Dual-Boot für den Fall, dass der Anwender doch etwas verwenden möchte, was nur unter Windows funktioniert.

Vorgehen

  • Mit garted-Live-CD Festplatte "aufräumen"
    • C-Partition verkleinern, D-Partition löschen, Restore-Partition löschen
    • Übrig bleibt C-Partition und eine leere erweiterte Partition (und eine Restore-Partition?)
  • openSUSE 13.2 32 Bit DVD einlegen und Setup
  • Anstelle der Voreinstellung btrs als Root-Dateisystem verwende ich ext4. Siehe dazu nächster Abschnitt.
  • Für die Home-Partition ein xfs-Dateisystem mit Verschlüsselung. Das vergebene Passwort muss dann immer während des Boot-Vorgangs eingegeben werden.
  • Nötige Anwender-Software nachinstallieren.

Fallstricke und Lösungen

Beim Installieren schlägt das openSUSE-Setup btrfs-Dateisystem vor. Dies habe ich auch so durchgeführt, um dann gegen Ende der Installation festzustellen, dass der grub-Bootloader nicht installiert werden kann.

Das openSUSE-Setup zeigt in diesem Fall nur eine generische Fehlermeldung.

Mit Hilfe dieser Anleitung RepairMBRafterWindowsinstall kann man versuchen, den grub per grub-install per Hand zu installieren und erhält "error: filesystem btrfs doesn't support blocklists." als Fehlermeldung. Weiter kam ich damit aber nicht.

Nach Internet-Recherche (z. B. Partition-Setup-for-Dual-Boot-with-OpenSUSE-13-2) auf gut Glück das Setup nochmal ausgeführt, aber diesmal ext4 statt btrfs ausgewählt. Damit hat die Installation des Bootloaders dann problemlos funktioniert.
